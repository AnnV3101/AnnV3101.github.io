10 частых ошибок в Python и как их избежать

📅 25 февраля 2026 ⏱️ 6 минут чтения

Даже опытные разработчики иногда попадают в эти ловушки. А новички — тем более. Вот 10 самых распространённых ошибок в Python.

1. Забыть двоеточие после if, for, while

Частая синтаксическая ошибка:

if x > 5 # Ошибка: пропущено :
    print("Больше")

Правильно:

if x > 5:
    print("Больше")

2. Неправильные отступы

В Python отступы важнее скобок. Смешивать табуляцию и пробелы нельзя.

def func():
    print("Привет") # 4 пробела
      print("Мир") # 6 пробелов — ошибка

3. Изменение списка во время итерации

numbers = [1, 2, 3, 4]
for n in numbers:
    if n % 2 == 0:
        numbers.remove(n) # Опасно!

Лучше создать копию: for n in numbers[:]:

4. Использование изменяемых объектов как значений по умолчанию

def add_item(item, lst=[]):
    lst.append(item)
    return lst

Список lst будет общим для всех вызовов. Правильно: lst=None.

5. Путаница между == и is

== проверяет равенство значений, is — что это один и тот же объект.

a = [1, 2, 3]
b = [1, 2, 3]
print(a == b) # True
print(a is b) # False

6. Забыть закрыть файл

Всегда используй with open() as f:, чтобы файл закрылся автоматически.

7. Ошибки при работе с кодировками

Указывай кодировку при открытии файлов: open('file.txt', encoding='utf-8').

8. Использование + для конкатенации строк в цикле

Это очень медленно. Используй ''.join(list).

9. Путаница в глобальных и локальных переменных

x = 10
def func():
    x = 5 # создаёт новую локальную x
    print(x)

Чтобы изменить глобальную, используй global x.

10. Игнорирование исключений

Голый except: ловит всё, включая Ctrl+C. Лучше указывать конкретные исключения.

Заключение

Большинство этих ошибок легко обнаруживаются, если использовать хороший редактор (VS Code, PyCharm) и линтеры (pylint, flake8).
